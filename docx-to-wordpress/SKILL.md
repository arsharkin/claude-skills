# docx-to-wordpress — Конвертация статей в WordPress HTML

Конвертирует .docx статьи в два HTML-файла: чистый (для вставки в WP) и на согласование (правки выделены жирным).

## Шаг 0: Определи цель

- Передан путь к **файлу** `.docx` → обработай один файл
- Передан путь к **папке** → найди все `.docx` в ней, обработай каждый
- Ничего не передано → спроси: «Укажи путь к файлу или папке»

## Шаг 1: Конвертация docx → markdown

Для каждого `.docx` запусти через Bash:
```bash
markitdown "путь/к/файлу.docx"
```
Результат — markdown-текст. Промежуточный `.md` файл на диск не пишем.

Если markitdown не найден: сообщи `pip install markitdown` и остановись.

## Шаг 2: Извлеки структуру

Из markdown определи:
- **Заголовок** — первая строка `# ...` → название статьи
- **Подзаголовки** — `## ...` → станут `<h2>`
- **Абзацы** — обычный текст → `<p>`
- **Списки** — `- ` → `<ul>/<li>`, `1. ` → `<ol>/<li>`
- **Футер** — блок начиная со слов «Уникальность» или «ключевые фразы» → не включать в HTML

## Шаг 3: Сгенерируй EXCERPT и META DESCRIPTION

- **EXCERPT** (~200–250 символов) — для поля «Цитата» в WordPress, суть статьи
- **META DESCRIPTION** (~150–160 символов) — для поля AIOSEO, содержит ключевое слово из заголовка

## Шаг 4: Найди ошибки русского языка

Проверь: пропущенные запятые при вводных словах («например», «таким образом», «конечно»),
несогласование падежей и числа, неверное слитное/раздельное написание, пропущенные предлоги.
Зафиксируй список — понадобится для файла на согласование.

## Шаг 5: Сформируй HTML

Правила (детали → `references/html-rules.md`):
- Теги только: `<p>`, `<h2>`, `<ul>`, `<ol>`, `<li>`, `<strong>`
- **Без пустых строк** между тегами
- Без `<!-- wp:... -->` — используется классический редактор WordPress
- Текст слово в слово из оригинала, не пересказывать

## Шаг 6: Запиши два файла

Имя = имя `.docx` без расширения. Папка = та же, где лежит `.docx`.

**`[название].html`** — чистая версия с применёнными правками (формат → `references/template-clean.md`)
**`[название] — на согласование.html`** — правки выделены `<strong>` (формат → `references/template-review.md`)
